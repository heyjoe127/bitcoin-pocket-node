Zeus wallet embeds LND. Connects to bitcoind over P2P. Full sovereign Lightning stack on a phone.

GrapheneOS killed it
GrapheneOS blocks child processes. LND runs as a separate binary. The OS terminates it immediately. Dead on arrival.
If you care about privacy enough to run GrapheneOS, you probably also care about running your own Lightning node. These goals were incompatible.

The Lightning safety model
If your counterparty broadcasts a revoked state while your phone is off, you have ~24 hours to respond with a justice transaction.

Phones die. Phones restart nightly. Phones get left in drawers.

You need a watchtower.
üè†
I already have one
LND on Umbrel at home. It has a built-in watchtower.

Problem: LND towers don't speak LDK.

Two different Lightning implementations.
Two different commitment formats.
No existing bridge.

Step 4

Build a watchtower bridge

1
Capture channel states from LDK
Hook into ldk-node's persistence layer. Every time a channel state updates, extract the commitment data.
2
Translate to LND's format
LND towers expect "justice blobs" in a specific binary format. Encode LDK's channel data into LND's JusticeKit structure.
3
Speak LND's wire protocol
LND towers use BOLT 8 with secp256k1 key exchange. No existing Rust library supports this. Had to implement the full Brontide handshake from scratch.
‚úì
Verified against live LND tower
Handshake, session creation, state updates. Justice blobs stored on the home Umbrel. All working.

One more thing

The tower is on .onion

The watchtower runs behind Tor on Umbrel. The phone needs to reach it.

üßÖ
Embedded Tor (Arti)
Tor's Rust implementation compiled into the app. 13 MB.

Phone connects directly to the tower's .onion address.
No Orbot. No proxy. No external app.

Because I needed to push 300 bytes to a watchtower.
